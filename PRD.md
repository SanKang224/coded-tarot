## 1. Problem (해결하려는 문제)

### 현재 타로 앱의 두 가지 결핍

**결핍 1 — 질문의 피상성**  
대부분의 타로 앱은 "그 사람과 잘 될까요?" 같은 모호한 질문을 그대로 받아 해석한다. 타로 리딩의 본질은 질문을 정확히 정의하는 과정에 있는데, 이 과정이 생략된다.

**결핍 2 — 몰입감 부재**  
기존 앱들은 화려한 카드 이미지와 UI로 채워져 있지만, 진짜 타로 리더에게 상담받는 긴장감·신비감·무게감이 없다.

### CODED-TAROT의 해법

MUD 게임 감성의 텍스트 터미널 인터페이스로 몰입감을 극대화하고, AI 마녀가 질문 유형을 판단·고도화한 뒤, 물리 셔플 알고리즘과 직접 카드 추출 경험을 통해 사용자가 진짜 타로 리딩에 참여하고 있다는 감각을 만든다.

## 2. Users (대상 사용자 / 페르소나)

### 주 타깃
- **연령:** 18~35세 여성
- **관심사:** 타로, 점술, 신비주의, 자기계발, SNS 감성 콘텐츠
- **행동 패턴:** 유료 타로 리더 예약은 부담스럽지만 타로에 진지하게 의미를 두는 사람.

### 페르소나

**페르소나 A — "진지한 고민러"**  
이직, 연애, 인간관계 등 실제로 고민이 생길 때마다 타로를 찾는다. 날카로운 해석을 원한다. 마녀 캐릭터의 직설적인 톤에 오히려 신뢰감을 느낀다.

**페르소나 B — "감성 수집가"**  
MUD 감성의 텍스트 인터페이스 자체가 신기하고 재밌다. 리포트를 스크린샷 찍어 SNS에 공유하거나 노션에 저장한다.

**페르소나 C — "레트로 게임 팬"**  
MUD, 텍스트 어드벤처 게임, 사이버펑크 미학에 공감한다. 타로를 게임처럼 즐기되 해석 결과는 진지하게 받아들인다.

### 브랜드 정체성
**"1-Bit Neon Witch"** — 1980년대 터미널 해킹 미학 + 마녀 오라클의 신비로운 권위.  
차갑다. 날카롭다. 위로하지 않는다. 진실만 말한다.

### 톤앤매너 (절대 규칙)

**허용 어미:** `~하라` / `~다` / `~이다` / `~인가` / `~것이다` / `~군`

**금지 어미 (영구 금지):** `~해요` / `~하세요` / `~습니다` / `~요` / `~죠` / `~할게요` / `~해주세요` / `~입니다`

**금지 표현:** "힘내세요", "잘 될 거예요", "응원해요", "안타깝네요" 등 공감/위로 표현 일체

## 8. 상세 유저 플로우 명세

### [Phase 0] 접속 시퀀스 (Boot Sequence)

모든 사용자(신규/기존)가 앱 최초 진입 시 보는 해킹 스타일 접속 로그:

```
- - - - - - - - - - - - - - - -
>> SIGNAL_DETECTED :: INITIATING_HANDSHAKE...
>> REROUTING: [랜덤IP] → [랜덤IP] → [랜덤IP]
>> BYPASS_01: PROXY_CHAIN_ALPHA...
   └ [OK]
>> BYPASS_02: FIREWALL_SECTOR_4F...
   └ [OK]
>> BYPASS_03: DARK_RELAY_NODE — ENCRYPTED...
   └ [OK]
>> LOCATION: UNRESOLVABLE — SIGNAL_MASKED
>> WARNING: 이 연결은 기록되지 않는다.
- - - - - - - - - - - - - - - -
[랜덤 인사말 중 하나]
```

**랜덤 인사말 풀 (매 접속마다 랜덤):**
- 마녀의 터미널에 접속했다.
- 카드는 준비되어 있다.
- 또 왔군.
- 무엇이 알고 싶은가.
- 카드는 거짓말을 하지 않는다.

**IP는 매 접속마다 랜덤 생성.** BYPASS 단계별 지연시간이 다르게 설정되어 (1초 → 1.2초 → 1.4초) 깊어지는 느낌을 준다.

---

### [Phase 1] 진입 게이트 (Entry Gate)

**14세 미만 접근 차단:**
- 생년월일 입력 없음 — `"14세 미만은 이용할 수 없다."` 고지 후 로그인으로 직행

**로그인 방식:** 카카오 / 구글 (터미널 스타일 메뉴 선택)
- 네이버: Supabase 미지원으로 제거
- 이메일: 구현됨, 현재 비활성화

**최초 로그인 완료 시 (최초 1회 한정):**
```
처음이군.
가방 안을 확인하라.
...
토큰 3개가 들어 있다.
언제부터 있었던 것인지는 알 수 없다.
```

---

### [Phase 2] 메인 메뉴

```
[Q] 질문   [T] 토큰
```

**인식 패턴:**
- 질문: Q / ㅂ / ㅃ / "질문" / "고민" / "타로" / "봐줘" 등
- 토큰: T / ㅅ / ㅆ / "토큰" / "충전" 등
- **토큰 충전은 질문 플로우 도중 어느 단계에서든 인식** (인증 전 단계 제외)

---

### [Phase 3] 질문 입력 및 분석

**본인 / 타인 확인:**
- 최초 질문 시 1회만 확인 (`identityConfirmed` 상태로 관리)
- 꼬리질문 시 재확인 없음
- 타인 리딩 시: `■ 마녀는 타인의 기록을 남기지 않는다.` 고지 후 진행

**AI 분석 (STEP 순서):**

```
STEP 0 (최우선): 타이밍 질문 감지
  "언제~될까 / 할까 / 가능할까" 패턴 → 무조건 TIMING

STEP 1: 리딩 타입 명확 → 경우 A (즉시 플랜 출력)
STEP 2: QUESTION/FLOW 선택 필요 → 경우 C (CHOICE 제시)
STEP 3: 컨텍스트 보충 필요
  → 본인 리딩: 경우 D (CONFIRM — AI 추론 항목 확인)
  → 타인 리딩: 경우 B (질문 1개)
```

**리딩 타입 3가지:**

| 타입 | 설명 | 카드 수 |
|------|------|---------|
| QUESTION | 구체적 YES/NO, A vs B, 단일 에너지 질문 | 1~3장 |
| FLOW | 기간별 흐름 (월/주/일 단위) | 2~6장 (최대 12) |
| TIMING | "언제~될까" 시점 질문 → 카드 숫자를 시간으로 해석 | 1장 고정 |

**CHOICE 포맷 (경우 C):**
```
■ [상황 진단]. 어떤 방식으로 볼 것인가?
1. 질문 — [예시]
2. 흐름 — [예시]
3. 다른 질문이 있다
```

**4회 차단 알고리즘:**
- AI가 경우 B (추가 질문)를 4회 연속 반환 시 → 글리치 셧다운 + 강제 리셋
- 토큰 차감 없음

**주제 전환 감지 (AI 기반):**
- 꼬리질문 완료 후 새 질문 입력 시, `/api/analyze`에 이전 주제 컨텍스트를 함께 전달
- Gemini가 STEP -1에서 NEW_SPREAD 여부 먼저 판단
- 감지 케이스: 주제 변경 / 같은 주제라도 대상 인물 변경 / 명시적 새 질문 선언
- NEW_SPREAD 반환 시 → 확인 요청, Y면 덱 리셋 후 저장된 질문 자동 재처리 (재입력 불필요)
- N이면 현재 덱 유지하고 꼬리질문으로 처리

---

### [Phase 4] 셔플 알고리즘 (The Witch's Shuffle)

1. **리플 셔플 + 역방향 혼합 (3회 반복)**
2. **컷 셔플 (3뭉치 분할 후 랜덤 재병합)**
3. **단면 정렬 검사 (Alignment Check):**
   - 정/역방향 비율 + 최대 연속 편향 검사
   - 불균형 감지 시 현재 상태에서 이어서 재셔플 (최대 4회)
4. 셔플 완료 후 ShuffleOverlay 애니메이션 + 정렬 검사 로그 출력

**꼬리질문 시 기존 덱 재사용** — 셔플 없음.

---

### [Phase 5] 카드 드로우 (The 78 Variables)

**CardGrid 표시:**
```
[C01] [C02] ... [C78]  ──▶ (1행 가로 스크롤)
```
- 각 카드의 Opacity 랜덤 (0.4~1.0)
- 노이즈 문자 (`░▒▓█∴∵`) 랜덤 프리픽스
- 이미 뽑힌 카드: 회색 + 취소선 (덱 위치 기반 + 카드 ID 기반 이중 비활성)
- 선택된 카드: `[1·C24]` 형식으로 순서 표시

**드로우 제한:**
- 스프레드당 최대 **15장** (세션 카운터로 추적)
- 잔여 드로우 표시: `>> 잔여 드로우 N/15`
- 15장 소진 시: `■ [SPREAD_EXHAUSTED]` → 다른 주제 입력 유도 → 전체 리셋 후 새 세션

**TIMING 모드:**
- 카드 1장 드로우 후 `/api/read`로 타이밍 힌트 포함 AI 해석 출력
- Major (0~21): 카드 번호 그대로 → N개월/N주
- Minor: 수트 내 위치 (Ace=1 ~ King=14)

---

### [Phase 6] 리포트 출력

**포맷 (포지션당):**
```
✦ [포지션명]
   "[포지션 질문]"
"선고 한 문장."
해석 한 문장.
- - - - - - - - - - - - - - - -
```

**종합 해석 (2장 이상 리딩 시):**
- 모든 포지션 완료 후 `/api/synthesis` 호출
- Gemini가 전체 카드를 보고 최종 판단 출력
- 선택지 리딩: 어떤 선택이 유리한지 직접 제시 (전부 좋거나 나쁘면 선택 기준 제시)
- 흐름 리딩: 전체 방향성 + 변곡점
- 일반 다중 포지션: 핵심 메시지

**토큰 차감 순서:**
- 해석 성공 후 차감 (실패 시 차감 없음)

**꼬리질문 처리:**
- 리딩 완료 후 화면의 CardReading 초기화 (이전 결과 가림)
- copy 스냅샷은 누적 유지 (`─────` 구분선으로 합산)

**복사 (`/copy`):**
- c / C / copy / /copy / 복사 / 추출 / 카피 등 다양한 입력 인식
- 클립보드에 전체 리딩 결과 복사 (꼬리질문 누적 포함)

## 9. 상점 및 비즈니스 모델 (The Altar)

**기본 모델:**
- 가입 시 최초 **3토큰** 지급 (1회 한정)
- 카드 1장 추출 = 1토큰 소비
- 토큰 0 시 추출 불가 → 충전 유도

**토큰 가격표:**

| 상품 | 토큰 | 가격 | 단가 |
|------|------|------|------|
| 소 | 3토큰 | 990원 | 330원/토큰 |
| 중 | 15토큰 | 4,450원 | 297원/토큰 |
| 대 | 30토큰 | 8,910원 | 297원/토큰 |

> 중·대 상품이 소 상품 대비 약 10% 저렴하지만, 이 사실을 상점에 명시하지 않는다.

**상점 진입:**
- 메인 메뉴 `[T]` 또는 질문 플로우 도중 "토큰" / "충전" 입력 시 즉시 이동

**이스터에그 — 비밀 코드:**
- 상점 화면 포함 어디에도 코드 입력창/안내 없음
- 메인 입력창에 `/code [비밀구문]` 입력 시에만 토큰 즉시 지급
- 인플루언서 콜라보 등 외부 경로로만 배포

## 11. 확정된 설계 결정 사항

| # | 항목 | 결정 내용 |
|---|------|----------|
| 1 | 카드 추출 방식 | 포지션 수만큼 한 번에 다중 선택 (1~6장) |
| 2 | 토큰 차감 단위 | 카드 1장 = 1토큰 |
| 3 | 세션당 최대 드로우 | 15장 (스프레드 전체 합산) |
| 4 | 꼬리질문 덱 처리 | 기존 덱 재사용, 재셔플 없음 |
| 5 | 카드 비활성 방식 | 덱 위치(drawnDeckIndices) + 카드 ID(drawnCardIds) 이중 추적 |
| 6 | 본인/타인 재확인 | 꼬리질문 시 생략 (identityConfirmed 플래그) |
| 7 | 주제 전환 감지 | AI(Gemini)가 NEW_SPREAD 신호 반환 — 주제·인물 변경 모두 감지, Y시 재입력 불필요 |
| 8 | TIMING 해석 | `/api/read` 호출, 타이밍 힌트(카드 번호 → 일/주/월) 포함 AI 해석 |
| 9 | copy 스냅샷 | 꼬리질문 포함 전체 누적, `─────` 구분선 |
| 10 | 세션 소진 후 처리 | 재셔플 아닌 새 주제 입력 유도 → 전체 리셋 |
| 11 | 토큰 충전 진입 | 어느 단계에서든 T/토큰/충전 입력 시 즉시 이동 |
| 12 | 관리자 계정 | is_admin=true 시 토큰 무한(∞) 표시, 차감 없음 |
